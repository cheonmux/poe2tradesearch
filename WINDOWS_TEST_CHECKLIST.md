# Windows 빌드 테스트 체크리스트 (배포 전 필수)

> 작성: 2026-06-12 (9차 세션).
> **중요**: 아래 코드 변경은 전부 macOS에서 논리/문법 검증만 됨. 실제 빌드·실행은 한 번도 안 됨.
> 집에서 Windows 빌드 후 이 목록을 1회씩 직접 확인해야 배포 안전. 통과하면 배포 OK.

## 빌드

```
msbuild Poe2TradeSearch.sln /p:Configuration=Release /p:Platform="Any CPU"
```

- [ ] **빌드 자체 통과** (컴파일 에러 0). 빌드 전 앱 종료 필수(.exe 잠금).
- [ ] 관리자 권한으로 실행 (핫키/단축키 기능).

## A. 9차 세션 — 자동 숨김 UX (4건)

### A-1. 새 검색 시 이전 타이머 무효화 (Methods.cs `mHideTimer?.Stop()`)
- [ ] 아이템 A Ctrl+C → 결과 뜸 → **~3초 뒤** 아이템 B Ctrl+C → **B 결과 뜨기 전에 창이 사라지지 않는지** 확인. (이전 버그: A 5초 타이머가 살아서 B 표시 전 숨음)

### A-2. 자동 숨김 시간 설정화 (`hide_delay`, 기본 5)
- [ ] 설정창(⚙) → "숨김까지 대기(초)" 입력칸 보임.
- [ ] 값을 10으로 변경 → 시세 표시 후 10초 뒤 숨는지.
- [ ] 값을 **0**으로 → 자동 숨김 **안 됨** 확인.
- [ ] 숫자만 입력되는지 (문자 차단).
- [ ] ⚠️ 설정창(모달) 여는 동안 메인 시세창이 숨을 수 있음 — 영향 있는지 관찰.

### A-3. 입력 필드 편집 중 숨김 보류 (`IsKeyboardFocusWithin`)
- [ ] 시세 뜬 후 input 숫자칸 클릭/편집 중 → **안 닫힘** 확인.
- [ ] 편집 끝내고 게임 클릭(포커스 이동) → 5초 뒤 정상 숨김.
- [ ] 마우스 hover 중 안 닫힘(기존 기능 유지) 확인.

### A-4. 창 위치 기억 (`Position` 필드)
- [ ] 창 옮김 → 종료(X→Yes) → 재실행 → **같은 위치** 복원.
- [ ] 창 옮김 → "최소화" 버튼/트레이 숨김 → 다시 표시 → 위치 유지.
- [ ] (멀티모니터) 다른 모니터로 옮겨도 복원되는지.
- [ ] 화면 밖 좌표 저장돼도 화면 안으로 보정되는지 (클램프).

## A'. 10차 세션 — rate limit 대기 UX (2건, Methods.cs `WaitForRateLimit`)

### A'-1. 대기 문구 변경 (선제적 throttle)
- [ ] 연타 검색으로 속도 조절 발생 시 `tkPriceInfo`에 **"요청 속도 조절 — N초 대기"** 표시 (기존 "거래소 혼잡" 아님).
- [ ] 차단(BlockedSeconds>0) 시엔 기존대로 "거래소 요청 제한 — N초 후 재개".

### A'-2. 대기 중 자동 숨김 보류 (`mHideTimer?.Stop()`)
- [ ] 속도 조절/제한 **대기 중에는 시세 창이 안 닫힘** (대기가 HideDelay보다 길어도).
- [ ] 대기 끝 → 결과 표시 → **그 시점부터** HideDelay초 뒤 정상 숨김.
- [ ] 대기 없는 일반 검색은 기존대로 결과 후 HideDelay초 숨김 (회귀 없음).

## B. 9차 세션 — 코드 점검 수정 (안정성)

### B-1. CancellationTokenSource Dispose (P-H1)
- [ ] 아이템 빠르게 연타 Ctrl+C (10~20회) → **크래시/예외 없는지**. (Dispose 후 워커 접근 시 ObjectDisposedException 위험 — 안 나야 정상)

### B-2. SetClipText 백그라운드 스레드 (P-H2)
- [ ] 검색 여러 번 후 앱 "최소화"/종료가 **즉시** 되는지 (포그라운드 스레드가 종료 막던 문제).

### B-3. mLockUpdatePrice volatile + RemoveHook + BeginInvoke 제거 (P-H5/M9/M11)
- [ ] 자동검색 카운트다운 표시 정상 (AutoSearchTimer — BeginInvoke 제거 후).
- [ ] 장시간 연타 검색 시 검색 잠금 꼬임 없는지.

### B-4. ★ API ID 검증 (S-H1) — 오탐 주의 ★ — ✅ 통과 (2026-06-13)
- [x] **일반 아이템 Ctrl+C → 검색 정상 동작** (오탐 0 확인).
- [x] 시세 결과의 매물들 정상 표시 (무작위 매물 다수 정상).
- [x] 아이템명 클릭 → 브라우저로 거래소 열림 (Process.Start 정상).
- [x] 실제 listing ID = 영숫자 10자리(예 `2KX2kzVeUk`) → 인젝션 문자 없어 검증과 충돌 없음.
- [ ] 화폐/보조젬도 정상 (exchange 경로 회귀 — 재확인 권장).

### B-5. catch{} → Debug.WriteLine (S-L8)
- [ ] Release 빌드에 **로그 파일 안 생김** 확인 (Debug.WriteLine은 Release서 자동 제거).
- [ ] (선택) Debug 빌드 + VS 출력창에서 실패 시 메시지 보이는지.

## C. 8차 세션 — 미검증분 (이전부터 밀린 것)

> PORTING_LOG 기준 아직 Windows 테스트 안 된 항목. 이번에 같이 확인 권장.

- [ ] **마법(magic) 아이템** Ctrl+C → 검색됨 (예: "치명적인 조율된 마법봉 - 악령술사"). 접두/접미 벗겨 베이스 재매칭.
- [ ] **보조 젬** Ctrl+C → ninja 시세 표시 (예: 바타이르의 복수).
- [ ] **rate limit throttle** — 연타 검색 → `.log`의 STATUS 확인, 30분 차단 미발생.
- [ ] **소수점 값 검색** (`치명타 확률 +2.17%` 등) 정상.
- [ ] **ES 자동체크** — `에너지 보호막 최대치 +64` 체크되는지 (미해결 보고 있었음).
- [ ] UI 버튼: "최소화"=트레이 숨김 / 오른쪽 위 X=종료 확인 팝업.
- [ ] TLS: 거래소/ninja HTTP 정상 (TLS1.2).
